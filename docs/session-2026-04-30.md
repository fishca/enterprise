## What landed (committable shape, modulo eval fix)

### 1. Bytecode self-sufficiency complete

`ibByteCode` is now a **pure template** — opcodes + constants + manifest. No live `ibValue*`, no back-pointer to compile-side, no compile-context anywhere in runtime structures.

Removed:
- `ibCompileCode::m_parent` field — parent chain lives only on `ibCompileModule::m_parentModule`.
- `ibByteCode::m_compileModule` back-pointer — gone.
- `ibRunContext::m_compileContext` — gone (debugger reads bytecode directly).
- `ibCompileContext::ibFunction::m_compileContext` — `functionContext` now under `unique_ptr` in `CompileFunction`, dies with the call.
- `stubCtx` hack in `GetFunction` synth — replaced by `m_bCodeRet` flag on `ibFunction`.
- Legacy `OPER_SET` emit at function entry — folded into `OPER_FUNC_PARAM` declarators.
- `reinterpret_cast<ibCompileContext*>` in `OPER_FUNC.m_param1.m_numArray` — gone.
- `OPER_CALL_M` opcode-cache patching (`m_param4` on first call) — never improved warm path, removed; bytecode is now never mutated at runtime.

`ibProcUnitEvaluate` owns its `ibCompileCode` via `unique_ptr<ibCompileCode>` — RAII cleanup, no manual `delete m_pByteCode->m_compileModule`.

### 2. Tape declarators (forward AOT compatibility)

Four new opcodes in `codeDef.h`:
- `OPER_FUNC_PARAM` — emitted after `OPER_FUNC` for each parameter (slot, byref, default-value descriptor).
- `OPER_FUNC_LOCAL` — emitted inline at variable-declaration site (`AddVariable` callsite in `CompileDeclaration`).
- `OPER_CTX_BEGIN` / `OPER_CTX_END` — emitted at `{ }` block boundaries.

All four are NOP at runtime (debugger skip-list updated). Parallel data structures (`m_listFunc`, `m_listLocals`, `m_listVar`) are still **canonical** — tape is a redundant forward-form for AOT walkers. AOT-phase will flip canonical: walk tape once on load, build runtime indices, drop the parallel writes from compile-time emit.

`OPER_FUNC` is now opcode-driven: handler sets `cRunContext.m_currentFunction` via `FindFunctionByEntry`. The three explicit `FindFunctionByEntry` calls at `OPER_CALL` / `CallAsProc` / `CallAsFunc` call sites are gone.

Cascade const-correctness through procunit / debugger / catalogObject / documentObject / chartOfAccountsObject / chartOfCharacteristicTypesObject / commonObject / constantObject / moduleManager / form.cpp.

One **documented lying const_cast remains**:
```cpp

```
DEF_VAR_CONST branch returns `const ibValue&` from `m_pByteCode->m_listConst`; ternary common type forces `ibValue&` so writes elsewhere compile. Programming convention: constants are never written. To fully remove, split into `read_var(x)` / `write_var(x)` — ~50 callsite touches. Not done.

### 4. ibByteBinder runtime binding session

```cpp
class ibByteBinder {
    ibByteBinder(const std::vector<ibByteBinding>& manifest, bool delta = true);
    void SetVar(const wxString& name, ibValue* value);
    bool IsDelta() const;       void SetDelta(bool);
    const std::vector<ibValue*>& GetSlots() const;
private:
    const std::vector<ibByteBinding>& m_manifest;  // borrowed reference
    bool m_delta;
    std::vector<ibValue*> m_slots;
};
```

`m_listRequiredBindings` on bytecode = AOT-serializable manifest (`name + clsid + kind`). Three kinds:
- `External` — accessed via `name.prop` only (`Metadata.Catalogs`).
- `Context` — accessed via `name.prop` only, no global prop visibility (e.g. `ThisObject.Code`).
- `ContextSelfRef` — props/methods globally callable bare (`Catalogs`, `MyMethod()` without `Manager.`). Pass-3 prop introspection at compile.

Pre-Execute validation in `procunit::Execute(bc, binder, retVal)`:
```cpp
for (size_t i = 0; i < manifest.size(); i++) {
    if (slots[i] == nullptr) Error("Required binding not provided: %s", name);
    if (manifest[i].m_clsid != 0 && val->GetClassType() != manifest[i].m_clsid)
        Error("Binding type mismatch for %s: expected %u got %u");
    m_pRefLocVars[i] = val;
}
```

Three Execute overloads on `ibProcUnit`:
- `Execute(bc, binder, retVal)` — primary, full form.
- `Execute(bc, binder)` — same, no retVal.
- `Execute(bc)`, `Execute(bc, delta)`, `Execute(bc, retVal, delta)` — legacy compatibility, build empty binder internally.

`bRunModule` and `bDelta` collapsed into a single `delta` flag.

`ibRuntimeModuleDataObject::m_binder` — descriptor owns binder, populated at `Compile()` from compile-side staging maps (`m_listExternValue`, `m_listContextValue`). `BindContextVariable` forwards to both compile-side AND runtime binder when binder is alive.

`bc.CreateBinder(delta)` — factory wrapping bytecode's manifest in empty binder.
`compileCode.CreateBinder(delta)` — factory + auto-fills from compile-side staging.

### 5. ibValue refinement

`ibValue::Clone()` virtual — simple types value-copy (`TYPE_BOOLEAN/NUMBER/STRING/DATE/NULL/EMPTY` return `*this`); ref/aggregate default returns `ibValue()` (undefined) — singletons / metadata gracefully refuse to clone, surfacing as empty rather than silent share. Subclasses override for their semantic.

`ibValue::GetValue() const` (was non-const).

`CopyValue` overload set:
- `CopyValue(ibValue&, ibValue&)` — mutable lvalue (legacy aliasing for `TYPE_OLE/ENUM/VALUE`).
- `CopyValue(ibValue&, ibValue&&)` — rvalue: store in local lvalue, delegate.
- `CopyValue(ibValue&, const ibValue&)` — const: `CopyValue(dst, src.Clone())` — never aliases const into mutable slot. Order matters (defined AFTER mutable + rvalue) to avoid C4717 self-recursion warning.

### 6. Symbol resolution refactor

`ibByteCode::FindVariable(name, contextVar, fullVisDepth) → ResolvedVar`
`ibByteCode::FindFunction(name, contextVar, fullVisDepth) → ResolvedFunc`

Wraps `ResolveVariable` / `ResolveFunction` with contextVar gating (returns empty if `contextVar=true` and `m_strContext.IsEmpty()` — Pass-2 top-level externs not treated as context-prop).

`ibCompileContext::FindVariable / FindFunction` collapsed to:
```cpp
// own m_listVariable (find_if + CompareString — case-insensitive)
// → m_parentContext walk
// → m_compileModule->m_cByteCode.FindVariable(name, contextVar, depth)
//   then synth via ibVariable / ibFunction ctor
```

Constructor symmetry — both directions:
- `ibVariable(name, ibByteCodeVarInfo&)` — synth from bytecode info.
- `ibFunction(name, ibByteFunction&)` — synth from bytecode function.
- `ibParamVariable(name, ibByteParam&)` — synth from bytecode param.
- `ibByteCodeVarInfo(const CompileVar&)` — templated, mirror compile→bytecode.
- `ibByteFunction(long lAddress, const CompileFn&)` — templated, mirror compile→bytecode (replaces old `MakeFunction` static factory).

All ctors use member-init-list (no default+assign).

### 7. Misc but important

- **Original case preserved** in `m_listVariable` / `m_listFunction` map keys (no `MakeUpper` in `PushVariable` / `PushFunction` / `CompileFunction`'s `strFuncName`). Lookups still case-insensitive via `find_if + stringUtils::CompareString`. Fixes `THISOBJECT/REFERENCE` uppercase regression in debugger watch's catalog object expansion.

When `OPER_RET` reads `*pRefLocVars[index1]`, deref of null → ECX=0. **Hypothesis B:** Outer Execute's overload resolution picks wrong path. `CompileExpression` line 1491:
```cpp

```
should resolve to `Execute(ibByteCode& bc, ibValue& retVal, bool delta=true)` — but `pvarRetValue` is `ibValue&` (function param of `CompileExpression`), and `false` is bool. Should match the overload that internally builds binder + calls 3-arg form. **But check** that build resolves it correctly and doesn't go to `Execute(bc, ibByteBinder&, ibValue&)` with `false` interpreted as something else.

**Hypothesis C:** Frame-array setup at CompileExpression:1496-1507 (`m_pppArrayList[1] = pRunContext->m_pRefLocVars`) happens AFTER outer Execute call but BEFORE inner Execute. Outer Execute might be re-Reset()-ing m_pppArrayList (line 68 in Reset()), wiping the override. Verify ordering — Reset deletes m_pppArrayList, then Execute re-allocates at line 1045-1058. CompileExpression's overrides at 1503/1506 happen AFTER outer Execute returns. So they apply to the array allocated by outer Execute. Then inner Execute (from caller's Evaluate at line 1410) uses this allocated+overridden array.
