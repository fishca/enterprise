### 3. Anonymous-namespace-in-header fix (ODR) ✅

Surfaced during split #1 verification.  `datavgen.h` had:

```cpp
namespace
{
    enum WalkFlags { Walk_All, Walk_ExpandedOnly };
    enum { SortColumn_None = -2, SortColumn_Default = -1 };
    class SortOrder { ... };
} // anonymous namespace
```

Anonymous namespace in a HEADER produces per-translation-unit types —
each `.cpp` that includes the header gets its OWN `WalkFlags` / `SortOrder`
with different mangled names.  Worked fine while only `datavgen.cpp`
referenced these types; broke at link time as soon as
`datavgen.paged.cpp` called `ibDataViewCtrl::GetRowByItem(item, WalkFlags)`
defined in `datavgen.cpp`:

```
LNK2019: unresolved external symbol
    ibDataViewCtrl::GetRowByItem(... `anonymous namespace'::WalkFlags)
```

Switched to named namespace `ibDataViewInternal` + `using` declarations
at file scope so external API stays unchanged.  Standard ODR fix.

## Decisions to remember

* **Anonymous namespace in headers is an ODR landmine.** Each TU sees
  its own type with different mangled symbol — works until cross-TU
  reference triggers a link error.  Use a named namespace +
  `using`-decls for header-visible "private" types.
* **`ibDataViewMainWindow` class lives in
  `datavgen.paged.private.h` now.**  Forward decl in `datavgen.h`
  stays small (public surface), full definition in the private
  header so paged code paths can call its methods.  `wxIMPLEMENT_*` /
  event-table macros are still single-instance in `datavgen.cpp`.
* **gtest scope today is intentionally narrow** — three sanity tests
  prove the scaffold compiles + the contract is exercisable.  Filter+
  sort coverage and frontend walker tests will need follow-on
  iterations.

## Late add — TableBox coordinator + single idle entry point + GUI-owned idle/refetch

Iterated user-driven refactors to centralise selection / refetch /

### Default RAM-base GetFeatures + Tree-RAM Get*Fetch

`ibValueModelRamTableBase::GetFeatures()` (`tableInfo.h:1283`) now
hosts the `RamFetch` flag default — three concrete overrides on
TabularSection / `ibValueModelTable` / `ibValueRecordSetObject` were
identical copies and got removed.  Symmetric default
`Tree | RamFetch` on `ibValueModelRamTreeBase::GetFeatures()`
(`tableInfo.h:2000`).  TabularSection extends via
`auto f = base::GetFeatures(); f.flags |= Filters | Sorting;`
(`tabularSection.h:47-55`) so its filter row + header-click sort UI
light up.

`ibValueModelRamTreeBase` had no `Get*Fetch` impls — added three

for empty parent), anchor cursor for Next/Prev (`tableInfo.h:2070+`).
`ibValueTreeNode::m_children` is private but RamTreeBase is friend
so direct access stays clean.

### TableBox.ApplyCurrentLine — single programmatic coordinator

`ApplyCurrentLine(line, focus = true)` (`tableBox.h:182`,
`tableBox.cpp:204+`) — all programmatic paths to
`m_tableCurrentLine` go through it: assigns m_tableCurrentLine +
`UnselectAllRows` + `Select(item)` + optional `SetCurrentItem` +
optional `EnsureVisible`.  `focus=false` for silent-track without
keyboard grab / scroll.  Selection script event is NOT fired here
(stub-line from `FindRowValue` has empty body fields, script
handler crashes on read — was tried, reverted after debugging).
User-click event stays inline in `OnSelectionChanged`.

Callsites:
- OnIdleSeed final, after the resolve chain.
- `SetPropVal eCurrentRow` (script `tableBox.CurrentRow = ...`) —
  was bare `m_tableCurrentLine = ...` with no ctrl drive.

### IsSortable on model + system-sort veto on GUI

`ibDataViewModel::IsSortable(col)` virtual (`tableView.h:573`,
default true).  `ibValueModel::IsSortable` overrides — filters out
system sorts (folders-on-top, register identity tail) and missing

`datavgen.cpp::ibDataViewHeaderWindow::OnClick` (~line 344): after

`tableBoxEvent.cpp::OnColumnClick` shrunk to designer-only

### Single idle entry point (control)

`EVT_IDLE` bind on `ibDataViewCtrl` removed, `OnIdleEvent` body
folded into `OnInternalIdle` (`datavgen.cpp:5524+`).  Order:
1. col-widths / dirty pass.
2. PagedBootstrap (if pending).
3. Top-up fill.
4. Anti-flicker Thaw (if PagedRefresh froze).
5. External hook (`m_idleHook`, see `SetIdleHook`).

`OnInternalIdle` fires every idle pass (independent of wx idle-
throttle), unlike `wxEVT_IDLE` event-chain — more reliable.

`ibDataViewCtrl::SetIdleHook(IdleHook)` is public
(`datavgen.h:929`) — external coordinator (TableBox) registers a
callback.  TableBox's `Bind(wxEVT_IDLE, OnIdle)` removed; replaced
by `SetIdleHook([this, ctrl] { this->OnIdleSeed(ctrl); })` in
`Create`.  Hook fires AFTER bootstrap inside the same idle pass —
buffer is settled by then.

### TableBox no longer tracks size

`m_dataViewSizeChanged`, `m_dataViewSize`, `OnSize` handler + Bind
— all removed (`tableBox.h`, `tableBoxEvent.cpp`).  Resize → refetch
lives entirely on the control (`m_pagedSizeChanged → idle fill`).
TableBox.OnIdleSeed is gated by `m_dataViewUpdated` (set in
OnUpdated).

### OnUpdated → SchedulePagedRefresh — gated by mutation indicator

`OnUpdated` (`tableBox.cpp:495`) calls
`dataViewCtrl->SchedulePagedRefresh()` ONLY when
`m_formOwner->GetCreatedValue()` or `GetChangedValue()` is non-
empty.  The Refresh action handler already routes via
`model.RefetchAll → notifier → SchedulePagedRefresh`; an
unconditional call from OnUpdated produced a double-refresh +
visible flicker (desktop inline-dispatch finishes cycle 1 before
cycle 2 → second freeze/wipe).

### Notifier-level Freeze/Thaw across reset

`ibGenericDataViewModelNotifier::BeforeReset` / `AfterReset`
(`datavgen.cpp:554+`) bracket `m_tableAreaWin->Freeze()` /
`Thaw()` around `Cleared()` so paint events between BeforeReset
and AfterReset cannot show the intermediate empty-tree state.
PagedRefresh's inner Freeze (`m_pagedFrozenForBootstrap`) nests
safely — wxFreeze is reference-counted, OnInternalIdle's Thaw
closes both levels.  The default wx-base impl was a no-op —
flicker on any RefetchAll was latent.

### Select(item) routes to bootstrap-restore for paged

`ibDataViewCtrl::Select(item)` (`datavgen.cpp:5682`) now first
stamps `m_pagedRestoreSelection` via `SetPagedRestoreSelection`
(new public method, `datavgen.paged.cpp:54+`) when the model is
paged, then runs the regular `GetRowByItem + SelectRow +
ChangeCurrentRow` path.  Covers both timings:
- Stub item + bootstrap pending → restore-channel (PagedBootstrap
  matches via `IsEqualTo` against the fresh batch).
- Row already in the tree → wxSelectionStore highlight.

`ApplyCurrentLine` simplified — just calls `Select(item)`, no
paged-vs-non-paged knowledge.

### OnAddRow / OnDeleteRow script events

New `m_eventOnAddRow` / `m_eventOnDeleteRow` (`tableBox.h:259+`).
Fire from:
- GUI-driven path — `OnItemStartInserting` / `OnItemStartDeleting`.
- Programmatic path — `OnIdleSeed`'s createdValue branch after
  `ApplyCurrentLine`, gated by a local `fromCreated` flag.

Persisted at the end of `LoadData/SaveData`.  Pre-existing forms
saved before this addition fail to load (no version tag,
`ibReader::r_stringZ` does not tolerate EOF) — user confirmed
"made fresh forms", legacy metadata needs re-save in designer.

## Decisions to remember

* **Single idle entry point.**  `OnInternalIdle` owns the whole
  idle flow: col-widths, paged bootstrap/fill/thaw, external
  hooks.  External coordinators (TableBox) plug in via
  `SetIdleHook` callback, not via the wxEVT_IDLE chain.
  Guaranteed order: ctrl state is settled before the hook fires.
* **Resize and refetch — control's job.**  TableBox doesn't track
  size at all.  `m_pagedSizeChanged → OnInternalIdle fill`.
* **Runtime-side refetch trigger — only on mutation indicator.**
  OnUpdated's SchedulePagedRefresh is gated by
  createdValue / changedValue.  Plain RefreshForm cycles without
  mutation must not wipe the buffer (the Refresh action itself
  goes through the notifier path; double-fire = flicker).
* **Single coordinator for current line.**  `ApplyCurrentLine` is
  the only programmatic route to `m_tableCurrentLine`.  User-click
  via `OnSelectionChanged` stays as the pull-side inline path so
  the Selection event fires only on real user interaction, not
  on stub-driven restore.
* **Select(item) self-routes for paged.**  The bootstrap-restore
  channel `m_pagedRestoreSelection` is stamped by `Select` itself
  when `model->IsPagedModel()`.  External code does not need to
  know paged details — just `ctrl->Select(item)`.
* **Notifier Freeze/Thaw is required by default.**  The wx-base
  impl leaves the reset window unfrozen; any RefetchAll could
  flicker.  `ibGenericDataViewModelNotifier`'s override closes it.
